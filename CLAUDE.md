# CP2077 100% Tracker — Project Context

## What This Is
A local HTML dashboard that reads your Cyberpunk 2077 save metadata and tracks completion across all content categories. Generated by `tracker_local.py`, saved to `tracker_dashboard.html`.

## Key Files
| File | Purpose |
|------|---------|
| `tracker_local.py` | Main script: quest catalog, save parsing, HTML template generation |
| `tracker_dashboard.html` | Generated output — open in browser |
| `read_inventory.py` | Reads sav.dat via CyberpunkPythonHacks to scan iconic weapon inventory |
| `tracker_weapons.json` | Written by read_inventory.py; weapon found/not-found status (true/false) keyed by `_weapon_*` IDs |

## Running
```bash
python tracker_local.py            # regenerate dashboard
python tracker_local.py --no-open  # regenerate without opening browser
python read_inventory.py           # scan save for iconic weapons → tracker_weapons.json
```

## Save Format
- Save root: `C:\Users\defes\Saved Games\CD Projekt Red\Cyberpunk 2077`
- Uses **latest** `metadata.*.json` by mtime
- `finishedQuests` is a **space-separated string** of completed quest IDs
- **CRITICAL**: Only **parent IDs** appear (e.g. `q104`, `q105`); individual phase IDs (e.g. `q104_01_sabotage`) do NOT appear in finishedQuests

## Quest ID Conventions
| Prefix | Category |
|--------|----------|
| `q0xx` | Main story parent groups (q000=prologue, q001=tutorial, q003–q005=Act1, q101–q116=Act2-3) |
| `q3xx` | Phantom Liberty main quests |
| `mq0xx` | Minor quests (NOT main story — e.g. mq011=Shoot to Thrill, mq012=Burning Desire) |
| `sq0xx` | Side quests |
| `sts_*` | Gigs (street stories) |
| `ma_*` | Map activities (cyberpsychos, NCPD crimes) |
| `mq3xx` | PL minor quests |
| `_weapon_*` | Pseudo-IDs for iconic weapon tracking (manual/inventory) |

## Phase ID Split Pattern
When quests appear separately in the game journal but share a parent:
- Display `id` = phase ID from modding wiki (e.g. `q105_dollhouse`)
- `check_id` = parent that appears in finishedQuests (e.g. `q105`)
- `quest_done()` uses `check_id` for the actual lookup

Example: Automatic Love chain — all 4 sub-quests use `check_id: "q105"` but have distinct display IDs.

## Wiki Links
- Only shown for: Main Story, Side Jobs, Phantom Liberty, PL Side, Minor Activities, Iconic Weapons
- Suppressed for: all Gig categories, Cyberpsychos, NCPD (fandom pages are empty stubs)
- Per-quest `wiki` field: `None`=auto-generate, `False`=suppress, `"slug"`=use this slug
- Hothead gig uses explicit `"wiki": "Hothead_(gig)"`

## CyberpunkPythonHacks Patch
```python
import cp2077chunk
cp2077chunk.DataChunkTableChunk.VALID_CAPACITY = (0x100, 0x200, 0x400)
```
Required for save v2.31 compatibility.

## Weapon Inventory Scan
Uses CRC32 hash lookup: `binascii.crc32(tweakdb_id.encode()) & 0xFFFFFFFF`
Scans every 4-byte window of the inventory node data for matching hashes.
